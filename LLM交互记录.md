# 我们和 LLM 的交互内容整理

## 1. 项目理解与任务核对

我们先核对了课程项目二的任务目标，确认这是一个基于 7 个分子描述符预测水溶解度 `logS` 的回归任务。

课程项目二里强调的核心点包括：
- 理解 7 个化学特征的物理意义
- 完整的数据分析、预处理、建模、评估流程
- 对比线性模型和神经网络
- 可视化预测结果
- 结合 LLM 辅助思考，但不能直接替代人工验证

我们在 Notebook 中把这些目标重新整理成了适合展示的建模流程，并强调了回归任务的本质，而不是分类任务。

## 3. 发现并修复的 bug

最初让AI生成的初模型结果出现过明显异常，尤其是线性基线在初始配置下 `RMSE` 非常大，不符合正常训练表现。

排查过程中重点检查了：
- 训练集和测试集是否划分合理
- 标准化是否只在训练集上 fit
- 目标值 `y` 的维度是否一致
- `RMSE` 与 `R2` 的计算是否正确
- 训练配置是否适合线性模型
- Notebook 单元执行顺序是否依赖前置变量

最后我们把主要问题定位为训练初始参数不合适，线性模型和神经网络采用了相同的参数，这无疑是不合理的，于是我们调整线性模型的参数，数据结果大幅提升

## 4. 预测准确率提升过程

为了提升预测准确率，我做了几轮逐步优化：

### 4.1 基础模型对比

先比较了线性基线和普通 MLP：
- 线性基线：`Test RMSE = 0.7351`
- MLP：`Test RMSE = 0.6913`

这说明神经网络已经比线性模型略好。

### 4.2 更稳健的数据划分

随后改成了 `70/15/15` 的训练/验证/测试划分，并用验证集选取最优 epoch。

结果进一步稳定：
- LinearBaseline(70/15/15)：`Test RMSE = 0.7420`
- MLPModel(70/15/15)：`Test RMSE = 0.6688`

### 4.3 二阶特征 + 更强 MLP

接着我们加入了：
- 二阶多项式特征
- 更深的 MLP
- GELU 激活
- Dropout
- AdamW 优化器
- HuberLoss / SmoothL1Loss
- 学习率调度器
- 梯度裁剪

这一步的 5-fold 结果进一步改善：
- 原 5-fold MLP：`CV RMSE Mean = 0.6478`
- TunedMLP + Poly2：`CV RMSE Mean = 0.5865`

RMSE 改善幅度约为 `0.0613`。

### 4.4 模型集成

最后我们又做了 5 个不同 seed 的增强模型集成，集成后的表现最好：
- 5 模型集成：`Test RMSE = 0.5600`
- `Test MAE = 0.4224`
- `Test R2 = 0.9754`

这也是目前 Notebook 里的最终最好结果。

## 5. 误差分析和特征重要性

为了进一步理解模型，我们还做了：
- 误差最大的样本分析
- 置换特征重要性分析

结果显示，影响最大的特征大致是：
- `MolWt`
- `LogP`
- `TPSA`

这与化学上对溶解度的直觉是一致的，说明模型主要抓住了分子量、疏水性和极性相关的信息。
